System Overview

This blog system is built with pure HTML, CSS, and JavaScript, requiring no server-side processing or build tools. It features a consistent design system, responsive layout, and modular components that make it easy to expand.

Key Features:

  • Mobile-first responsive design
  • Unified color scheme with CSS variables
  • Dark/light mode support
  • Post filtering by category
  • Modular structure for easy expansion
  • Interactive components (Call Tracker, Journal)

Adding New Pages

To add a new page to the blog system, follow these steps:

Method 1: Copy an Existing Page

  1. Copy an existing HTML file (like smell_of_wet_soil.html) to use as a template
  2. Rename the file to match your new content (e.g., my-new-post.html)
  3. Update the <title> tag and metadata
  4. Replace the content in the <main> section with your new content
  5. Update the category and publication date
  6. Add a link to your new page in index.html

Method 2: Create From Scratch

Updating the Navigation

After creating a new page, make sure to update the navigation in all HTML files to include the new page. Look for the <nav> section in each file.

Modifying the Design

CSS Variables

The design system uses CSS variables for consistent styling. You can modify these in the <style> section of each HTML file: 

:root {
    --primary-color: #3b82f6;    /* Main accent color */
    --secondary-color: #6366f1;  /* Secondary accent color */
    --text-color: #1f2937;       /* Main text color */
    --bg-color: #f9fafb;         /* Background color */
    --card-bg: #ffffff;          /* Card background color */
    --border-color: #e5e7eb;     /* Border color */
    --font-family: 'Inter', system-ui, -apple-system, sans-serif;
}

Dark Mode

Dark mode is implemented using media queries and CSS variables. You can customize the dark mode appearance: 

@media (prefers-color-scheme: dark) {
    :root {
        --text-color: #f3f4f6;
        --bg-color: #111827;
        --card-bg: #1f2937;
        --border-color: #374151;
    }
}

Tailwind Customization

For more advanced customization beyond what CSS variables offer, you can modify Tailwind classes directly in the HTML. Common Tailwind utility classes used in the system:

  • container - Contains content to a max-width
  • mx-auto - Centers content horizontally
  • px-4 py-6 - Padding on x-axis (4) and y-axis (6)
  • text-* - Text sizes (xs, sm, base, lg, xl, 2xl, etc.)
  • font-* - Font weights (normal, medium, semibold, bold)
  • bg-* - Background colors
  • text-* - Text colors
  • dark:* - Dark mode variants

Customizing Functionality

JavaScript Overview

The scripts.js file contains all the JavaScript functionality for the blog. It's organized into sections:

  • Common Functionality - Mobile navigation, dark mode, etc.
  • Blog Post Filtering - Category and search filtering
  • Theme Controls - Color and font customization
  • Call Tracker - Call tracking functionality
  • Metacognition Journal - Journal functionality

Adding New Functionality

To add new JavaScript functionality:

  1. Add a new section to scripts.js with your function
  2. Add a condition in the DOMContentLoaded event to initialize your function
  3. Add the necessary HTML elements to your page

Example of adding new functionality:

// In scripts.js
document.addEventListener('DOMContentLoaded', function() {
    // Existing code...
    
    // Add your condition
    if (document.querySelector('.your-new-feature')) {
        initializeYourNewFeature();
    }
});

// Add your new function
function initializeYourNewFeature() {
    // Your functionality code here
}

Modifying Existing Functionality

To modify existing functionality, find the relevant section in scripts.js and make your changes. Each major component has its own initialization function that you can modify.

Working with Media

Images

Add images to your content using standard HTML: 

<img src="path/to/your/image.jpg" alt="Description of the image" class="max-w-full h-auto rounded my-4">

Note: Avoid using base64 encoded images as they make the HTML file excessively large.

Videos

For embedding videos, especially MP4 files: 

<video controls playsinline webkit-playsinline class="max-w-full my-4 rounded">
    <source src="your_video_url" type="video/mp4">
    Your browser does not support the video tag.
</video>

The playsinline and webkit-playsinline attributes ensure compatibility with both Chrome and Safari.
